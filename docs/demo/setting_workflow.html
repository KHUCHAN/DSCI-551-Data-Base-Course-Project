Hybrid-AML Shell Setup Workflow

Docker Desktop + reset_pipeline.sh + PostgreSQL + Apache AGE

1. Reset and Start Environment

Bring up a clean AGE-enabled PostgreSQL container from the repository root.

flowchart LR A["Open Docker Desktop"] --> B["Clone repo and
enter project root"] B --> C["Run
./scripts/reset_pipeline.sh"] C --> D["Pull
apache/age image"] D --> E["Remove old container
or volume if needed"] E --> F["Start fresh AGE-enabled
PostgreSQL container"] F --> G["Wait for PostgreSQL
readiness"]

2. Load Data and Build Graphs

Generate the synthetic CSVs, load relational tables, then project into AGE.

flowchart LR H["Generate synthetic
CSV data"] --> I["Create relational
schema"] I --> J["Load customers,
accounts, transactions"] J --> K["Enable Apache AGE
extension"] K --> L["Build baseline graph
aml_graph"] L --> M["Build tuned graph
aml_graph_tuned"] M --> N["Verify PostgreSQL
row counts"]

3. Demo Queries and Benchmark

Use the ready environment to show the AML detections and optional performance comparison.

flowchart LR O["Environment ready
for live demo"] --> P["Run
smurfing_detection.sql"] P --> Q["Run recursive SQL
cycle detection"] Q --> R["Run baseline Cypher
cycle detection"] R --> S["Run tuned Cypher
cycle detection"] S --> T["Optionally run
benchmark_cycle_queries.py"]
